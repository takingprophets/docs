## Overview

**Prophet Model** is TakingProphets' core entry framework rendered as a single indicator. It integrates four components into one workflow: Fair Value Gaps (the imbalance zone that defines the entry area), CISD (the Change in State of Delivery line that confirms the setup), EPE (the Engineered Paired Entry — the refined entry level within the FVG), and Limit levels (TP, BE, and SL targets). When all four elements align, the indicator draws each component on the chart and can fire an alert on CISD confirmation.

Two optional tables enhance the trading session workflow: a Relationships table that shows contextual market data, and a Checklist table that tracks which setup conditions have been met. The Prophet Model is the complete execution layer — from setup identification through limit order placement.

## How We Use It

Prophet Model is the team's primary execution indicator. It runs on the 1-minute chart during live sessions and draws the complete setup: the FVG defines the zone, the CISD fires the alert and confirms the setup direction, the EPE marks the limit entry price within the zone, and the limit levels set TP, BE, and SL with a single glance. The CISD alert is the trigger — when it fires inside a killzone with HTF confluence, the team evaluates the setup immediately using the checklist table to verify all conditions. The model is designed so that every element needed to enter and manage the trade is visible on one indicator at the same time.
