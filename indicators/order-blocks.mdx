## Overview

**Order Blocks** identifies two complementary ICT concepts: order blocks and breaker blocks. An order block is the last bearish candle before a bullish impulsive move (bullish OB) or the last bullish candle before a bearish impulsive move (bearish OB). These zones represent where institutional buy or sell orders were placed to initiate the move, and price frequently returns to them for continuation entries.

A breaker block forms when an order block is subsequently violated — the institutional orders that defined it have been broken, and the zone "flips" to act as resistance (former support broken) or support (former resistance broken). Breaker blocks are often the cleaner re-entry zones after a liquidity sweep because they mark the precise level where the prior institutional order flow was exhausted.

The indicator draws each block as a colored zone and optionally shows breaker blocks in a separate color when they form.

## Settings

**OB Sensitivity** — `High` | `Medium` | `Low` (default: `Medium`)

Controls the detection threshold for order block qualification. `High` detects more order blocks with looser criteria; `Low` detects fewer but more structurally significant blocks. `Medium` is the balanced default for most timeframes.

## How We Use It

Order Blocks is the team's standard order block reference overlay. The sensitivity setting is kept at `Medium` for daily use — `High` generates too much noise on fast 1-minute charts, and `Low` misses many valid intraday blocks. Breaker blocks are enabled and watched carefully: when a bullish order block is broken to the downside, the resulting bearish breaker is often the exact re-entry zone for shorts if price returns to it. The combination of an order block entry aligned with a Fair Value Gap or CISD on the same level is the highest-conviction setup.
